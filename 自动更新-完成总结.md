## 📱 用户体验

### 自动更新流程
```
用户打开应用
    ↓
后台检查更新
    ↓
发现新版本 → 弹出通知
    ↓
自动下载（后台）
    ↓
下载完成 → 询问是否安装
    ↓
用户点击"立即重启"
    ↓
自动安装并重启
    ↓
✅ 更新完成！
```

### 手动检查更新
```
菜单：文件 → 检查更新
    ↓
开始检查
    ↓
有更新 → 下载并提示
无更新 → 提示"已是最新版本"
```

## 🔄 发布新版本流程

每次发布更新的完整流程：

```powershell
# 1. 更新代码
# ... 修改你的代码 ...

# 2. 更新版本号
npm version patch  # 1.0.0 -> 1.0.1（小修复）
# 或
npm version minor  # 1.0.0 -> 1.1.0（新功能）
# 或
npm version major  # 1.0.0 -> 2.0.0（大更新）

# 3. 重新打包
npm run electron:build:win

# 4. 发布到GitHub
.\发布.ps1 -Version "1.0.1" -Title "v1.0.1 - 更新说明"

# 5. 完成！用户会自动收到更新
```

## ⚠️ 重要提醒

### 必须做的事
1. ✅ 修改GitHub配置（owner和repo）
2. ✅ 创建GitHub仓库
3. ✅ 发布时上传 `.exe` 和 `.yml` 两个文件
4. ✅ Release Tag必须与版本号一致（v1.0.0 对应 1.0.0）

### 注意事项
1. **latest.yml 很重要**
   - 这是更新配置文件
   - 每次打包都会重新生成
   - 必须和安装包一起上传

2. **版本号规则**
   - package.json: `1.0.0`
   - Release Tag: `v1.0.0`
   - 文件名: `AI插件应用 Setup 1.0.0.exe`
   - 三者必须匹配！

3. **仓库必须公开**
   - 自动更新需要Public仓库
   - Private仓库需要额外配置token

## 💡 常见问题

### Q: 现在能用自动更新吗？
A: 还不能！需要先：
1. 创建GitHub仓库
2. 修改配置文件中的owner和repo
3. 发布到GitHub

### Q: 开发模式能测试更新吗？
A: 不能。开发模式自动更新被禁用。需要打包后测试。

### Q: 如何测试自动更新？
A: 
1. 发布v1.0.0
2. 安装到电脑
3. 修改代码，发布v1.0.1
4. 打开v1.0.0，会收到更新提示

### Q: 用户必须联网吗？
A: 是的。检查更新和下载都需要网络。但Ollama本地模型可以离线使用。
