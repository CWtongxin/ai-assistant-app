# 📦 发布到GitHub脚本

## 方法一：使用PowerShell脚本（推荐）

### 1. 安装GitHub CLI
```powershell
# Windows (使用winget)
winget install GitHub.cli

# 或访问 https://cli.github.com/ 下载安装
```

### 2. 登录GitHub
```powershell
gh auth login
# 选择 GitHub.com
# 选择 HTTPS
# 按提示完成登录
```

### 3. 初始化Git仓库（如果还没有）
```powershell
git init
git add .
git commit -m "Initial commit"
```

### 4. 关联GitHub仓库
```powershell
# 替换为你的仓库地址
git remote add origin https://github.com/你的用户名/ai-assistant-app.git
git branch -M main
git push -u origin main
```

### 5. 运行发布脚本
```powershell
# 创建v1.0.0 Release
.\发布.ps1 -Version "1.0.0" -Title "v1.0.0 - 初始版本"
```

## 方法二：手动发布（无需安装工具）

### 步骤：

1. **打开浏览器**
   访问：`https://github.com/你的用户名/ai-assistant-app/releases/new`

2. **填写发布信息**
   - Tag version: `v1.0.0`
   - Release title: `v1.0.0 - 初始版本`
   - Description: 填写更新说明

3. **上传文件**
   点击"Attach binaries..."，选择以下文件：
   - `release/AI插件应用 Setup 1.0.0.exe`
   - `release/latest.yml`

4. **发布**
   点击 "Publish release"

## 方法三：使用发布脚本（自动化）

### 创建发布脚本

保存以下内容为 `发布.ps1`：

```powershell
param(
    [string]$Version = "1.0.0",
    [string]$Title = "v$Version",
    [string]$Description = "AI助手 v$Version"
)

Write-Host "🚀 开始发布 v$Version..." -ForegroundColor Green

# 检查文件是否存在
$Installer = "release\AI插件应用 Setup $Version.exe"
$LatestYml = "release\latest.yml"

if (-not (Test-Path $Installer)) {
    Write-Host "❌ 错误：找不到安装包 $Installer" -ForegroundColor Red
    exit 1
}

if (-not (Test-Path $LatestYml)) {
    Write-Host "❌ 错误：找不到 latest.yml" -ForegroundColor Red
    exit 1
}

Write-Host "✅ 文件检查通过" -ForegroundColor Green
Write-Host "📦 安装包: $Installer"
Write-Host "📄 配置文件: $LatestYml"

# 使用GitHub CLI发布
if (Get-Command gh -ErrorAction SilentlyContinue) {
    Write-Host "📤 正在创建GitHub Release..." -ForegroundColor Yellow
    
    gh release create "v$Version" `
        --title "$Title" `
        --notes "$Description" `
        $Installer `
        $LatestYml
    
    if ($LASTEXITCODE -eq 0) {
        Write-Host "✅ 发布成功！" -ForegroundColor Green
        Write-Host "🌐 访问: https://github.com/$(gh repo view --json owner,name -q '.owner.login + "/" + .name')/releases/tag/v$Version"
    } else {
        Write-Host "❌ 发布失败" -ForegroundColor Red
    }
} else {
    Write-Host "⚠️  未检测到GitHub CLI，请手动发布：" -ForegroundColor Yellow
    Write-Host "1. 访问: https://github.com/你的用户名/ai-assistant-app/releases/new"
    Write-Host "2. Tag version: v$Version"
    Write-Host "3. 上传文件:"
    Write-Host "   - $Installer"
    Write-Host "   - $LatestYml"
    Write-Host "4. 点击 Publish release"
}
```

### 使用方法：
```powershell
# 发布v1.0.0
.\发布.ps1 -Version "1.0.0" -Title "v1.0.0 - 初始版本"

# 发布v1.0.1
.\发布.ps1 -Version "1.0.1" -Title "v1.0.1 - Bug修复"
```

## 📋 发布检查清单

发布前确认：

- [ ] 已修改 `electron/main.js` 中的GitHub配置
- [ ] 已修改 `package.json` 中的GitHub配置
- [ ] 已成功打包应用
- [ ] `release/latest.yml` 文件存在
- [ ] `release/AI插件应用 Setup 1.0.0.exe` 文件存在
- [ ] GitHub仓库已创建

## 🔄 发布新版本流程

每次发布新版本的完整流程：

```powershell
# 1. 更新版本号
npm version patch  # 或 minor, major

# 2. 打包
npm run electron:build:win

# 3. 获取新版本号
$Version = node -p "require('./package.json').version"

# 4. 发布
.\发布.ps1 -Version $Version -Title "v$Version - 更新说明"
```

---

## ⚠️ 注意事项

1. **版本号必须匹配**
   - package.json 中的 version: `1.0.0`
   - GitHub Release Tag: `v1.0.0`
   - 文件名: `AI插件应用 Setup 1.0.0.exe`

2. **latest.yml 很重要**
   - 这个文件包含版本信息
   - 自动更新功能依赖这个文件
   - 每次打包都会重新生成

3. **Release必须公开**
   - 仓库必须是 Public
   - Release不能是 Draft
   - 文件必须作为附件上传
