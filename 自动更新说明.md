# 🔄 自动更新功能说明

## ✅ 已实现的功能

### 1. 启动时自动检查更新
- 应用启动时会自动检查是否有新版本
- 如果有更新，会在后台自动下载
- 下载完成后弹出提示，询问是否立即重启安装

### 2. 手动检查更新
- 菜单栏：**文件 → 检查更新**
- 点击后会检查是否有新版本

### 3. 更新通知
- **发现新版本** - 弹出通知，后台下载
- **下载进度** - 控制台显示下载进度
- **下载完成** - 弹出对话框询问是否立即安装
- **更新错误** - 记录错误日志

## 📋 配置自动更新

### 方式一：使用GitHub Releases（推荐）

#### 1. 创建GitHub仓库
```bash
# 在GitHub上创建新仓库
# 例如：https://github.com/your-username/ai-plugin-app
```

#### 2. 修改配置

**文件：`electron/main.js`**
```javascript
autoUpdater.setFeedURL({
  provider: 'github',
  owner: 'your-username',  // 改为你的GitHub用户名
  repo: 'ai-plugin-app',   // 改为你的仓库名
  private: false
})
```

**文件：`package.json`**
```json
"publish": {
  "provider": "github",
  "owner": "your-username",  // 改为你的GitHub用户名
  "repo": "ai-plugin-app",   // 改为你的仓库名
  "releaseType": "release"
}
```

#### 3. 发布新版本
```bash
# 1. 更新版本号
npm version patch  # 或 minor, major

# 2. 打包并发布
npm run electron:build:win

# 3. 在GitHub创建Release
# - 访问 https://github.com/your-username/ai-plugin-app/releases
# - 点击 "Create a new release"
# - Tag version: v1.0.1
# - 上传打包的文件：
#   - AI插件应用 Setup 1.0.1.exe
#   - latest.yml（自动生成，很重要！）
```

---

### 方式二：使用通用HTTP服务器

#### 1. 修改配置

**文件：`electron/main.js`**
```javascript
autoUpdater.setFeedURL({
  provider: 'generic',
  url: 'https://your-server.com/updates/'
})
```

**文件：`package.json`**
```json
"publish": {
  "provider": "generic",
  "url": "https://your-server.com/updates/"
}
```

#### 2. 上传更新文件到服务器
将以下文件上传到你的服务器：
- `AI插件应用 Setup 1.0.1.exe`
- `latest.yml`（版本信息文件）

## 🚀 发布更新流程

### 完整步骤

1. **更新版本号**
```bash
npm version patch    # 1.0.0 -> 1.0.1
# 或
npm version minor    # 1.0.0 -> 1.1.0
# 或
npm version major    # 1.0.0 -> 2.0.0
```

2. **打包应用**
```bash
npm run electron:build:win
```

3. **创建GitHub Release**
   - 访问仓库的Releases页面
   - 创建新Release，标签为 `v1.0.1`
   - 上传文件：
     - `AI插件应用 Setup 1.0.1.exe`
     - `release/latest.yml`

4. **用户收到更新**
   - 用户启动应用时自动检查
   - 或手动点击"文件 → 检查更新"
   - 后台下载完成后提示安装

## 📝 版本管理建议

### 语义化版本控制

- **主版本号（major）** - 不兼容的API修改
- **次版本号（minor）** - 向下兼容的功能性新增
- **修订号（patch）** - 向下兼容的问题修正

### 示例

```
1.0.0 - 初始版本
1.0.1 - 修复bug
1.1.0 - 新增功能
2.0.0 - 重大更新
```

## 🔍 调试自动更新

### 开发模式
开发模式下自动更新会被禁用，控制台会显示：
```
开发模式，跳过自动更新
```

### 查看日志
在打包后的应用中，可以通过以下方式查看更新日志：

**Windows日志位置：**
```
%APPDATA%/ai-plugin-app/log.txt
```

### 常见错误

1. **无法连接到更新服务器**
   - 检查GitHub仓库配置是否正确
   - 确认Release已发布
   - 确认latest.yml文件存在

2. **版本号不正确**
   - 确保package.json中的version已更新
   - 确保Release标签与版本号一致

3. **下载失败**
   - 检查网络连接
   - 确认Release附件已正确上传

## 💡 最佳实践

### 1. 测试更新
- 发布前先内部测试
- 确认更新流程正常
- 验证安装后数据不丢失

### 2. 更新说明
- 每次发布都写清楚更新内容
- 告知用户需要做什么
- 提醒可能的破坏性变更

### 3. 频率控制
- 不要过于频繁地推送更新
- 重要bug修复立即发布
- 功能更新可以累积后发布

### 4. 版本回退
- 保留旧版本下载链接
- 如果新版本有严重问题，可以快速回退
